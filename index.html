Cuando mi e-commerce se rompió en mobile

Por Tomas Enrique Bruland · Mayo 2026 · HTML & CSS

Contexto

Durante el último mes estuve desarrollando ShopEasy, una tienda online estática hecha con HTML y CSS. El objetivo era crear una página de productos con encabezado, grilla de tarjetas y carrito lateral, para usarla como portfolio personal.

El Problema

Dos días antes de la entrega, un compañero revisó el sitio desde su celular y el layout estaba completamente roto: las tarjetas se salían de la pantalla y el menú tapaba el contenido.

La causa fue que usé medidas fijas en píxeles (width: 900px) en todos los componentes, sin ningún media query. Nunca había probado en mobile durante el desarrollo.

Acciones Tomadas

1. Diagnóstico: Usé las DevTools de Chrome con el simulador de dispositivos móviles para confirmar el problema.

2. Rama de trabajo: Creé una rama separada para no afectar la versión publicada:

git checkout -b fix/responsive-layout

3. Refactorización del CSS: Reemplacé medidas fijas por unidades relativas y agregué media queries:

/* ANTES */
.product-grid {
  width: 900px;
}

/* DESPUÉS */
.product-grid {
  width: 100%;
  flex-wrap: wrap;
}

@media (max-width: 768px) {
  .product-grid {
    flex-direction: column;
  }
}

4. Pull Request: Pedí al compañero que revisara desde su celular antes de aprobar los cambios.

5. Merge: Una vez aprobado, hice el merge a main y GitHub Pages actualizó el sitio automáticamente.

Aprendizajes

  • Mobile-first desde el inicio: es más fácil escalar de chico a grande que al revés.
  • Usar las DevTools siempre: el simulador de mobile debería estar abierto desde el primer día.
  • Trabajar con ramas: permite experimentar sin romper lo que ya funciona.
  • Compartir avances temprano: el feedback antes vale más que la revisión al final.

Reflexión: Feedback Radicalmente Sincero

Mi compañero fue directo: "Esto no funciona en mobile para nada." Mi primera reacción fue defensiva, pero luego entendí que ese feedback honesto era exactamente lo que necesitaba.

Aprendí a separar la crítica al código de una crítica personal. El código estaba roto; yo tenía la capacidad de arreglarlo. Hoy prefiero un comentario incómodo a tiempo que un elogio que no me ayuda a mejorar.
